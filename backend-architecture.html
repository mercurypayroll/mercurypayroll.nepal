Multi-Tenant Data Architecture
Single PostgreSQL database serving all client companies using Row-Level Security (RLS). Each row carries a tenant_id — queries are auto-filtered so Company A can never see Company B's data.

Database schema — key tables
-- Every table has tenant_id + RLS policy
CREATE TABLE employees (
  id          UUID PRIMARY KEY,
  tenant_id   UUID NOT NULL REFERENCES tenants(id),
  name        VARCHAR(200) NOT NULL,
  pan         VARCHAR(20),
  ssf_no      VARCHAR(30),
  basic_salary NUMERIC(12,2),
  department  VARCHAR(100),
  joined_date_bs DATE,
  status      VARCHAR(20) DEFAULT 'active',
  created_at  TIMESTAMPTZ DEFAULT NOW()
);

-- RLS policy — auto-filters every query
ALTER TABLE employees ENABLE ROW LEVEL SECURITY;
CREATE POLICY tenant_isolation ON employees
  USING (tenant_id = current_setting('app.current_tenant')::UUID);

-- Payroll table
CREATE TABLE payroll_runs (
  id          UUID PRIMARY KEY,
  tenant_id   UUID NOT NULL,
  month_bs    VARCHAR(7),   -- e.g. '2081-12'
  total_gross NUMERIC(15,2),
  total_tds   NUMERIC(15,2),
  total_ssf   NUMERIC(15,2),
  total_net   NUMERIC(15,2),
  status      VARCHAR(20),
  run_by      UUID REFERENCES users(id),
  run_at      TIMESTAMPTZ
);

Security Architecture
Enterprise-grade security — critical for financial payroll data handling PAN numbers, bank accounts, and salary information.
🔐
Authentication & authorization
✓JWT tokens with 24hr expiry + refresh tokens
✓Role-based access control (RBAC) — Admin, HR, Finance, Employee
✓Multi-factor authentication (TOTP/SMS OTP)
✓Session invalidation on password change
✓OAuth2 support for Google/Microsoft SSO
🏰
Data isolation (multi-tenancy)
✓PostgreSQL Row-Level Security on all tables
✓tenant_id injected at middleware level — not trusting client
✓Separate Redis namespaces per tenant
✓S3 bucket policies restrict file access by tenant
✓Database backups encrypted per tenant
🔒
Data encryption
✓TLS 1.3 for all data in transit
✓AES-256 encryption for PAN numbers at rest
✓Bank account numbers stored encrypted
✓Password hashing with bcrypt (cost factor 12)
✓Database encryption at rest (AWS RDS / DO)
🛡️
Nepal compliance & audit
✓Immutable audit log — every payroll action recorded
✓Who ran payroll, when, what changed — full history
✓IRD-compatible data retention (7 years)
✓GDPR-style data export and deletion per company
✓Rate limiting prevents brute-force attacks
