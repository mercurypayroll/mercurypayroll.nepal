🚀 Hosting Guide

Everything you need to get Mercury Payroll live online — from free static hosting to full production VPS deployment. Follow the steps below from top to bottom.

Step 1 — Choose your hosting method
FREE · 2 minutes ▲
Vercel (Recommended)
Best for the HTML frontend. Free SSL, CDN, and custom domain. No server needed.
1
Go to vercel.com — sign up free
2
Click "Add New Project"
3
Drag & drop the mercury-payroll folder
4
Click Deploy — done in 60 seconds
5
Add your .com.np domain in settings
FREE · GitHub 🐙
GitHub Pages
Host directly from GitHub. Great for version control and collaboration.
1
Create account at github.com
2
New repository → upload files
3
Settings → Pages → main branch
4
Live at yourusername.github.io/mercury
5
Connect .com.np domain in DNS
PRO · Full stack 🖥️
VPS / DigitalOcean
For full backend with real database, payroll engine, and payment APIs.
1
Get droplet at digitalocean.com (~$6/mo)
2
Install Node.js, PostgreSQL, Nginx
3
Upload files via Git or FTP
4
Point domain DNS to server IP
5
Install free SSL via Let's Encrypt
Step 2 — Deploy with Vercel (quickest path)
💡 This gets your Mercury Payroll site live in under 2 minutes — no coding required. Just drag, drop, deploy.

Option A — Via Vercel website (no coding needed)

1. Open: https://vercel.com 2. Click "Sign Up" → use Google or GitHub account 3. Click "Add New..." → "Project" 4. Click "Browse" → select your mercury-payroll folder 5. Leave all settings as default → click Deploy 6. ✅ Your site is live at: https://mercury-payroll-xyz.vercel.app

Option B — Via Vercel CLI (terminal)

# Install Node.js first from: https://nodejs.org # Then open terminal / command prompt: npm install -g vercel # Install Vercel CLI once cd mercury-payroll # Go to your project folder vercel # Login with email → confirm → deploy # Output: # ✅ Deployed to: https://mercury-payroll.vercel.app

After buying domain, add these DNS records at your registrar:

# For Vercel hosting — add these DNS records: Type Name Value A @ 76.76.21.21 CNAME www cname.vercel-dns.com # For GitHub Pages — add these DNS records: Type Name Value A @ 185.199.108.153 A @ 185.199.109.153 CNAME www yourusername.github.io

⏱️ DNS changes take 1–24 hours to propagate across Nepal ISPs.

Step 5 — Adding features later
➕

New module

Create a new .html file in /modules, add a nav link in index.html
🎨

Change colours

Edit :root CSS variables at the top of any file — 5 seconds
📊

New tax slab

Find the slabs[] array in tds-wizard.html and update numbers
💳

New payment gateway

Copy a tab in payment-gateway.html, rename, update API code
🏦

New bank

Add a new .bank-item div in the bank grid section
👤

New employee field

Add a .form-group div to the employee form in payroll-software.html
🌐

Real backend

Connect forms to Node.js/Express API — replace static data with fetch() calls
🔁

Re-deploy

After any change: run vercel again or push to GitHub — auto-deploys in 30s
